# 认知觉醒:开启自我改变的原动力-周岭
 
## 第一章 大脑 一切问题的起源
- 立足长远，主动走出舒适区；
- 为潜在的风险克制自己，为可能的收益延时满足；
- 保持耐心，坚持做那些短期内看不到效果的“无用之事”；
- 抵制诱惑，面对舒适和娱乐时，做出与其他人不同的选择……

焦虑的几种形势
1. 完成焦虑
2. 定位焦虑
3. 选择焦虑
4. 环境焦虑
5. 难度焦虑 （有些书就是很难读，有些文章就是很难写，有些知识就是很难懂，有些技能就是很难学……真正能让你变强的东西，其核心困难是无法回避的，不下决心与之死磕，始终在周围打转，时间越长越焦虑）

### 焦虑真正的根源
- 焦虑并不完全源于我们的主观意识，而是来自我们大脑的生理结构。
- 急于求成，想同时做很多事；
- 避难趋易，想不怎么努力就立即看到效果。

看清背后的机理并设法改变
- 克制欲望，不要让自己同时做很多事；
- 面对现实，看清自己真实的能力水平；
- 要事优先，想办法只做最重要的事情；
- 接受环境，在局限中做力所能及的事；
- 直面核心，狠狠逼自己一把去突破它


- 很多时候，我们对困难的事物缺乏耐心是因为看不到全局、不知道自己身在何处，所以总是拿着天性这把短视之尺到处衡量，以为做成一件事很简单
- 事实上，如果我们能了解一些事物发展的基本规律，改用理性这把客观之尺，则会极大地提升耐心。

复利效应显示了价值积累的普遍规律：前期增长非常缓慢，但到达一个拐点后会飞速增长。

怎样拥有耐心？
- 首先，面对天性，放下心理包袱，坦然接纳自己。 
  - “你看，我身体里那个原始人又出来了，让他离开丛林到城市生活，确实挺不容易的，要理解他
- 其次，面对诱惑，学会延迟满足，变对抗为沟通
  - 暂时忍耐一下，先做重要的事情，之后会有半小时或一小时的时间专门玩手机，想怎么玩都行
- 最后，面对困难，主动改变视角，赋予行动意义
  - 比如当我们清楚了阅读的本质和意义，就可能放下手机，主动拿起书本；
  - 当我们明白了深度学习的意义，就可能放弃听书、速读，转而开始精读和输出；
  - 当我们明白了运动真正的好处，就可能告别慵懒，主动坚持锻炼



当程序员在工作中突然没有任务可做时，可以考虑以下几种方式来合理利用时间：

1. 自我学习：利用这段时间来学习新的编程语言、框架或技术。阅读相关的技术博客、文档，观看在线教程或视频，扩展自己的技术知识和技能。

2. 项目优化：回顾已经完成的项目，寻找改进的空间。可以考虑进行代码重构、性能优化、界面改进等工作，以提高项目的质量和用户体验。

3. 开发工具和环境的改进：审视自己的开发环境，看看能否改进工作流程和效率。可以尝试寻找新的开发工具、插件或配置，以提升开发效率和舒适度。

4. 参与开源项目：贡献自己的力量给开源社区，可以选择一个与自己技术背景相关的开源项目，提交补丁、修复问题或参与讨论，积累经验和扩大人脉。

5. 技术调研：了解当前热门的技术趋势和发展方向，包括人工智能、大数据、区块链等。可以进行相关的调研和探索，以便将来在工作中有所应用。

6. 文档编写和知识分享：将自己的经验和知识整理成文档或博客，分享给团队或社区。编写技术教程、指南或演示文稿，可以帮助他人学习和理解相关的技术领域。

7. 与团队成员合作：与同事进行交流和合作，了解他们正在进行的项目或任务。你可以主动提供帮助，或者请求参与其他项目，以充实自己的工作内容。

8. 提升软技能：程序员不仅需要具备技术能力，还需要不断提升自己的软技能。可以学习项目管理、沟通技巧、团队协作等方面的知识，以更好地适应工作环境。

无论选择哪种方式，重要的是保持积极主动的态度，利用这段时间来提升自己的技能和知识，为将来的工作做好准备。
